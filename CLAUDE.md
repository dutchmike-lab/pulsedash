# Marketing Dashboard

Company-wide marketing dashboard for three brands: **Remodeling Concepts** (remodelingconcepts.net), **Ryann Reed** (ryannreed.com), and **Wood Lane Golf & Social**.

## Architecture

This project uses the WAT framework (Workflows, Agents, Tools):
- `tools/` — Python scripts that pull data from each API
- `workflows/` — Markdown SOPs for each data source
- `marketing-dashboard.html` — Self-contained HTML dashboard with placeholder data
- `.env` — All API keys and credentials (gitignored)

## Data Sources

| Source | API | Status |
|--------|-----|--------|
| Google Analytics (GA4) | GA4 Data API | Needs credentials |
| Google Ads | Google Ads API | Account exists, not active |
| Meta Ads | Marketing API | Account exists, not active |
| YouTube | YouTube Data API v3 | 2 separate channels |
| Instagram | Instagram Graph API | 2 accounts via Meta Business |
| Email | Constant Contact API v3 | Active |
| Forms — Gravity Forms | WordPress REST API | Active on WordPress sites |
| Forms — GoHighLevel | GHL API | Active |
| Forms — Custom Node.js | GHL API (submissions route to GHL) | Active, hosted on Vercel |

## Tech Stack

- Python 3.x for data tools
- google-analytics-data, google-ads, facebook-business SDKs
- Self-contained HTML dashboard (Chart.js for sparklines)
- Data flows: APIs -> Python tools -> JSON -> Dashboard

## Conventions

- All secrets in `.env`, never hardcoded
- Tool scripts are standalone and testable
- Each tool outputs JSON to stdout or writes to `.tmp/`
- Dashboard reads from a single `data.json` file generated by a master pull script
